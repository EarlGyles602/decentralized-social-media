# Smart Contract Implementation for Decentralized Social Media Platform

## Overview

This pull request introduces the core smart contract infrastructure for a decentralized social media platform built on Stacks blockchain. The implementation provides comprehensive functionality for content ownership, monetization, and community-driven moderation.

## Contracts Implemented

### 🔐 Content Ownership Contract (`content-ownership.clar`)

**Primary Functions:**
- **Content Creation & Ownership**: Users can create content with cryptographic proof of authorship
- **Direct Monetization**: Creators receive tips and subscriptions directly from their audience
- **Licensing System**: Built-in content licensing with public, commercial, and exclusive options
- **Access Control**: Premium content access based on subscriptions or ownership
- **Revenue Tracking**: Comprehensive earnings tracking for creators

**Key Features:**
- Content registry with IPFS/decentralized storage references
- STX-based tipping system with platform fee structure
- Subscription management with time-based access control
- Multi-tier licensing (public, commercial, exclusive)
- Creator earnings dashboard and analytics

**Public Functions:**
- `create-content()` - Create new content with ownership proof
- `tip-creator()` - Send STX tips to content creators
- `subscribe-to-creator()` - Purchase creator subscriptions
- `purchase-license()` - Buy content usage licenses
- `access-content()` - Access premium content with permission checks
- `update-platform-fee()` - Platform fee management (owner only)

### 🏛️ Moderation Governance Contract (`moderation-governance.clar`)

**Primary Functions:**
- **Community Moderation**: Decentralized content moderation through community voting
- **Reputation System**: Merit-based moderation privileges and voting weights
- **Appeal Process**: Fair dispute resolution with jury-based appeals
- **Platform Governance**: Democratic decision-making for platform policies
- **Moderator Management**: Automatic moderator status based on reputation

**Key Features:**
- Content reporting and community-driven moderation decisions
- Reputation-weighted voting system
- Multi-tier appeal process with random jury selection
- Governance proposal creation and voting
- Automatic moderator status assignment based on reputation thresholds

**Public Functions:**
- `report-content()` - Submit content for community moderation
- `vote-on-report()` - Vote on moderation decisions
- `submit-appeal()` - Appeal moderation decisions
- `jury-vote-appeal()` - Jury voting on appeals
- `create-proposal()` - Create platform governance proposals
- `vote-on-proposal()` - Vote on governance proposals
